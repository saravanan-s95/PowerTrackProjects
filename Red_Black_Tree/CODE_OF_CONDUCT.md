# Code of Conduct

## Our Commitment

This is a technical learning project. Everyone who interacts with this repository — contributors, reviewers, and maintainers — is expected to engage respectfully and in good faith.

We do not have a large community to govern. What we do have is code to write and a standard of behavior to hold.

---

## Expected Behavior

- Be direct and technical. Disagree with code, not with people.
- Give constructive feedback. If you point out a problem, suggest a fix or ask a question.
- Acknowledge your own mistakes and incomplete understanding. We are all still learning.
- Be honest about the origin of your contributions, including AI-generated content.

---

## Unacceptable Behavior

- Personal attacks, insults, or harassment in issues, PRs, or discussions.
- Submitting code you do not understand and cannot explain, regardless of how it was generated.
- Deliberately introducing bugs or breaking changes without disclosure.
- Dismissing contributions based on the contributor's background rather than the technical merit.

---

## Enforcement

This is a small project. Enforcement is simple: behavior that violates the above will result in a warning, then removal of contribution access if it continues. There is no formal appeals process — maintainers have final say.

---

## Scope

This code of conduct applies to all project spaces: GitHub issues, pull requests, discussions, and any communication directly related to this project.

---

## Contact

If you have a concern, open a private GitHub issue or email the maintainer directly. We will respond as soon as reasonably possible.
